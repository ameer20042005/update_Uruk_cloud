# تحديثات أوروك السحابي - ameer20042005/update_Uruk_cloud 

هذا المستودع مخصّص لنشر تحديثات برنامج **أوروك (Uruk POS)** فقط.
لا يحتوي على الكود المصدري — فقط ملفات المثبّت (Releases) التي يقوم
التطبيق بفحصها وتنزيلها تلقائياً.

> ⚠️ يجب أن يبقى هذا المستودع **عاماً (Public)** حتى يستطيع التطبيق فحص
> الإصدارات عبر واجهة GitHub API بدون رمز مصادقة.

---

## كيف يعمل التحديث

1. التطبيق يستعلم عن آخر إصدار من:
   `https://api.github.com/repos/ameer20042005/update_Uruk/releases/latest`
2. يقارن رقم الوسم (tag) مع إصدار التطبيق الحالي.
3. إن وُجد إصدار أحدث، ينزّل ملف `.exe` المرفق ويشغّله، ثم يُغلق التطبيق
   ليُكمل المستخدم خطوات التثبيت.

---

## كيف ترفع إصداراً جديداً (Release)

### 1) جهّز ملف المثبّت
في مشروع البرنامج:
1. ارفع رقم الإصدار في `pubspec.yaml` (مثال: `version: 1.0.2+1`)
   **و** في `installer/uruk_setup.iss` (`#define AppVersion "1.0.2"`).
   يجب أن يتطابق الرقمان.
2. شغّل `installer/build_installer.bat`.
3. الناتج: `installer/output/uruk-pos-setup-1.0.2.exe`.

### 2) أنشئ Release على هذا المستودع
**عبر الموقع:**
1. افتح: `https://github.com/ameer20042005/update_Uruk/releases/new`
2. **Choose a tag** → اكتب رقم الإصدار (مثل `1.0.2` أو `v1.0.2`) ثم
   اختر *Create new tag*.
3. **Release title**: اكتب عنواناً (مثل `الإصدار 1.0.2`).
4. **Describe this release**: اكتب ملاحظات التحديث — ستظهر للمستخدم داخل
   التطبيق.
5. اسحب ملف `uruk-pos-setup-1.0.2.exe` إلى منطقة **Attach binaries**.
6. اضغط **Publish release**.

**أو عبر سطر الأوامر (GitHub CLI):**
```bash
gh release create 1.0.2 ^
  "installer/output/uruk-pos-setup-1.0.2.exe" ^
  --repo ameer20042005/update_Uruk ^
  --title "الإصدار 1.0.2" ^
  --notes "إصلاحات وتحسينات في الأداء."
```

## قواعد مهمة

| القاعدة | التفصيل |
|---------|---------|
| **رقم الوسم (tag)** | يجب أن يكون أعلى من الإصدار الحالي (مقارنة x.y.z). بادئة `v` تُحذف تلقائياً. |
| **الملف المرفق** | يجب أن ينتهي بامتداد `.exe`. يُؤخذ أول ملف `.exe` يُعثر عليه. |
| **خصوصية المستودع** | يجب أن يبقى Public. |
| **ملاحظات الإصدار** | وصف الـ Release يظهر للمستخدم — اكتبه بالعربية بوضوح. |

---

## مثال على رقم الإصدار

- التطبيق الحالي: `1.0.1` → ترفع Release بوسم `1.0.2` → يظهر التحديث ✅
- التطبيق الحالي: `1.0.2` → ترفع Release بوسم `1.0.2` → لا يظهر تحديث (مطابق).
